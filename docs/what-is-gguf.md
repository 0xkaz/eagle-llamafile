# GGUF とは何か

> モデルを「1ファイルで持ち運べる」ようにするためのフォーマット

---

## 一言で言うと

GGUF（GPT-Generated Unified Format）は、LLM の**重み・設定・メタデータをすべて1つのバイナリファイルにまとめた形式**。
モデルを動かすのに必要なものが全部入っているため、ファイルを置くだけで推論エンジンが読み込める。

---

## 生まれた背景

LLM の重みは元々 PyTorch（`.pt` / `.bin`）や SafeTensors（`.safetensors`）で配布されていた。
これらは**学習・研究向け**のフォーマットで、推論だけしたいユーザーには不便だった：

- 複数ファイルに分割されている（`pytorch_model-00001-of-00003.bin` など）
- Python / PyTorch 環境が必須
- モデルの設定（レイヤー数、語彙サイズ等）が別の JSON ファイルに分離している
- 量子化（圧縮）が標準化されていない

GGUF はこれらの問題を解決するために **llama.cpp プロジェクト**が策定した。

### 前身：GGML → GGUF への移行（2023年8月）

```
GGML 形式（〜2023-08）
  └─ 後方互換性がなく、フォーマット変更のたびに全ファイル再配布が必要
        ↓ 廃止
GGUF 形式（2023-08〜現在）
  └─ 拡張可能なメタデータ設計、後方互換性あり
```

## ファイル構造

```
┌─────────────────────────────────────────┐
│  GGUF ファイル (.gguf)                   │
│                                         │
│  [ヘッダー]                             │
│    マジックナンバー: "GGUF"              │
│    バージョン番号                        │
│    テンソル数・メタデータ数              │
│                                         │
│  [メタデータ]                           │
│    モデル名・アーキテクチャ名            │
│    コンテキスト長・レイヤー数            │
│    トークナイザー語彙・設定              │
│    量子化方式                           │
│    ← すべて key-value 形式で格納        │
│                                         │
│  [テンソルデータ]                       │
│    各レイヤーの重み（量子化済み）        │
│    アライメント調整済み                  │
└─────────────────────────────────────────┘
```

「モデルカード（設定）」と「重み（データ）」が**1ファイルに同居**しているのが最大の特徴。

---

## 量子化とは

元の重みは FP32（32ビット浮動小数点）で学習される。推論時はこれを**より少ないビット数に丸める（量子化）**ことでサイズと速度を改善できる。

| 量子化方式 | ビット数 | 7B モデルのサイズ | 品質 |
|-----------|---------|-----------------|------|
| F32 | 32bit | ~28 GB | 完全精度 |
| F16 | 16bit | ~14 GB | ほぼ完全 |
| Q8_0 | 8bit | ~7.5 GB | 高品質 |
| **Q5_K_M** | **5bit** | **~5 GB** | **高品質（推奨）** |
| **Q4_K_M** | **4bit** | **~4 GB** | **良好（標準推奨）** |
| Q3_K_M | 3bit | ~3 GB | 許容範囲 |
| Q2_K | 2bit | ~2.5 GB | 劣化大 |

`K` は K-quants（グループ単位の非均一量子化）、`M` は Medium（精度とサイズのバランス型）を意味する。

**本プロジェクトの方針:** Q4_K_M（標準）/ Q5_K_M（品質重視）/ Q8_0（高精度）の3バリアントを提供する。

## なぜ llamafile に必要か

llamafile は **GGUF ファイルを実行ファイルに「貼り付ける」** 形で単一バイナリを作る。

```
llamafile の構造：

┌──────────────────────────────────────┐
│  llamafile 実行ファイル              │
│                                      │
│  [llama.cpp 推論エンジン]            │  ← C++ バイナリ
│  [Cosmopolitan Libc]                 │  ← クロスプラットフォーム層
│  ─────────────────────────────────   │
│  [GGUF モデルファイル]               │  ← ここに重みが入っている
└──────────────────────────────────────┘

実行すると自動で Web UI (port 8080) + OpenAI 互換 API が起動
```

GGUF が「自己記述的な1ファイル」であることが、llamafile の「1ファイルで動く」という思想と完全に一致している。

## 変換フロー（本プロジェクトでの使い方）

```
HuggingFace から weights をダウンロード
（.safetensors / .bin 形式）
        ↓
convert_hf_to_gguf.py（llama.cpp 付属）
        ↓
GGUF FP16（フル精度）
        ↓
llama-quantize（llama.cpp 付属）
        ↓
GGUF Q4_K_M / Q5_K_M / Q8_0
        ↓
llamafile bundle ツール
        ↓
.llamafile（単一実行ファイル）
```

## 対応ツール・エコシステム

| ツール | GGUF の使い方 |
|--------|--------------|
| **llama.cpp** | GGUF を直接ロードして推論 |
| **llamafile** | GGUF を実行ファイルにバンドル |
| **Ollama** | GGUF をモデルとして登録・管理 |
| **LM Studio** | GGUF をGUIで選択・実行 |
| **Jan** | GGUF をローカルで実行 |
| **Open WebUI** | llama.cpp 経由でGGUF を使用 |

HuggingFace では GGUF ファイルをそのままダウンロードして上記ツールで使える。

---

## まとめ

| 特徴 | 内容 |
|------|------|
| 単一ファイル | 設定・語彙・重みが全部入り |
| 量子化対応 | FP32〜2bit まで同一形式で扱える |
| 後方互換性 | メタデータの追加が破壊的変更にならない |
| ライブラリ非依存 | Python / PyTorch 不要で推論可能 |
| エコシステム | llama.cpp / Ollama / LM Studio 等すべてが対応 |

**GGUF があれば llamafile 化は「あと一歩」。**
本プロジェクトはその「あと一歩」を、UAE/Saudi モデルについて実施することを目的としている。
